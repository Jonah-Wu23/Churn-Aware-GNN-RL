# 项目演进文档 (Project Evolution Document)

## 📋 文档说明
本文档记录了基于流失博弈的社区微公交调度系统（Mobi-Churn）的开发及优化历程。按时间顺序总结了系统从最初的架构设计到性能优化、Bug 修复及训练环境稳定的关键节点。

---

## 🕒 时间轴与关键里程碑

### 阶段一：架构设计与基准建立 (2026-01-12)
*   **模型定义**：确立了以"乘客流失（Churn）"为核心信号的 GNN-RL 调度框架。
*   **双阶段仿真**：定义了 Stage 1 (Gym 极速训练) 与 Stage 2 (SUMO 真实性验证) 的架构。
*   **课程学习**：设计了 L0-L4 的阶梯式难度方案，并引入 $\rho$（有效负荷率）作为晋级指标。
*   **详见文档**：`流失驱动的社区微公交动态调度架构设计_合并版.md`

---

### 阶段二：深度训练性能优化 (2026-01-14)
针对原始模型在显存利用率（仅 800MB）和训练时长（25小时+）上的瓶颈进行了深度重构：
*   **模型层替换**：将自定义卷积替换为高性能的 `TransformerConv`，推理成本降低 80%。
*   **批量反向传播**：将逐样本 Q 值计算改为批量堆叠后一次性 backward，显存利用率显著提升且计算图更稳定。
*   **混合精度引入**：启用 AMP (Autocast) 进一步压缩显存并加速浮点运算。
*   **成果**：显存占用达到 20GB 级别，训练时间缩短至 10-16 小时，GPU 利用率提升至 80% 以上。
*   **详见文档**：`docs/项目演进文档/DQN_OPTIMIZATION_COMPLETE.md`

---

### 阶段三：Structural Unserviceable 统计问题诊断 (2026-01-14)
*   **现象**：发现日志中 `structural_unserviceable` 数值（1715/2026）异常偏高。
*   **诊断**：确认该数值并非 Bug，而是课程学习阶段订单池的全局死单统计。由于图为弱连通但非强连通（4个SCC），跨SCC请求无有向路径。
*   **结论**：实际可达率仍保持在 99.9% 以上，不影响训练效果。可选择增加 `neighbor_k` 或保留最大 SCC 来实现强连通。
*   **详见文档**：`docs/项目演进文档/structural_unserviceable.md`、`docs/项目演进文档/structural_unserviceable_diagnosis.md`

---

### 阶段四：训练稳定性与 NaN Loss 专项修复 (2026-01-15)
针对 L2/L3 阶段频繁出现的 `NaN Loss` 导致模型崩溃的问题进行了根本性修复：
*   **特征净化**：识别出图结构不连通产生的 `inf` 值是罪魁祸首。修改 `get_feature_batch`，对 ETA、乘客计数等特征进行硬性封顶 (`MAX_TIME_VAL = 36000.0`)。
*   **Loss 守卫**：在 DQN 优化器中添加 `torch.isnan(loss)` 检查，异常步自动跳过，确保权重不被污染。
*   **Bug 修复（关键）**：发现并修复了课程学习中 `max_requests` 覆盖 Bug。现在配置文件中的 `1500` 单设置能正确生效，极大缓解了训练压力。

### 阶段五：训练数据分割与车队规模调整 (2026-01-15)
*   **时间划分**：实现 30% 训练集与 70% 评估集的时间轴分割（前18天用于训练，后43天用于评估）。
*   **运力匹配**：将车辆数从 50 调整至 51 辆（以 `max_requests: 1500` 为基准，确保系统运力盈余约 10%）。

### 阶段九：L1 无有效动作逃逸修复 (2026-01-22 - 迭代2)
*   **问题症状**：
    *   L1 的 `rho_window_mean` 长期约 0.20，无法达到 0.4 过关阈值。
    *   频繁出现短 episode（约 600 步）且 `mask_valid_ratio=0.0`，`done_reason` 为 `unknown`。
    *   "无有效动作"路径直接 `done=True` 且 `reward=0`，绕过终止惩罚。
*   **修复内容**：
    *   无有效动作时必须触发终止惩罚，并计入 `episode_return`。
    *   `done_reason` 全链路可追踪，不再出现 `unknown` 漏网。
    *   统计口径包含 `waiting_timeouts` 与 backlog（`waiting_remaining/onboard_remaining`）。
    *   新增 `_compute_terminal_backlog_penalty` 统一计算 backlog 归一化惩罚。
    *   新增 `_attach_env`，L3 phase 切换时刷新 env 与图缓存，并同步 `env_cfg`。
*   **涉及文件**：`src/train/dqn.py`、`src/train/runner.py`、`src/env/gym_env.py`

### 阶段十：事件链修复与面板指标增强 (2026-01-22 - 迭代3)
*   **问题背景**：训练中出现 `effective_event_rate=0`、`churn_prob_mean=0`、`served=0` 的空转现象。

*   **面板诊断指标补充**：`current_time`、`ready_vehicles`、`event_queue_len`、`env_steps`。

### 阶段十一：潜势差分 Shaping 全链路 (2026-01-22 - 迭代4)
*   **目标**：打通潜势差分 shaping 全链路，保证 L1/L2 训练可解释、不返工。
*   **关键变更**：
    *   **统一 backlog 标量**：`waiting_remaining + onboard_remaining`。
    *   **统一 lost_total 口径**：`waiting_churned + onboard_churned + structural_unserviceable`。
    *   **潜势定义**：`phi = backlog + lost_weight * lost_total`，before/after 快照同源。
    *   **shaping 机制**：`reward_potential_shaping_raw = alpha * (phi_before - phi_after)`。
*   **报警与取证（RewardHackDetector）**：
    *   不变量检查覆盖潜势一致性、reward_total 对账、lost_total 口径、低熵警告。
    *   持久 alerts ring buffer，自动 dump 到 `reports/debug/potential_alerts/`。
*   **新增验证工具**：`scripts/smoke_test_potential.py`、`tests/test_potential_shaping.py`

### 阶段十三：L2 训练稳定性修复 (2026-01-22 - 迭代6)
*   **目的**：修复 L2 训练不稳定与评估 gate 失真问题，避免 no_valid_action 早终止。
*   **关键修复**：
    1. **训练阶段 gate 来源可配置**：新增 rho_gate_source / stage_rho_gate_source，支持 train/eval/max；L2 默认改为使用训练 rho_window_mean 作为过关判据。日志新增 rho_source 标注。
    2. **no_valid_action 根因处理**：action_mask 全 False 时，强制加入/放开 current_stop 作为 noop fallback。step 支持 wait 动作（action==current_stop 且不在邻接边中），并推进到下一事件时间。
    3. **回放缓冲区容量修复**：允许 noop fallback 增加 1 个动作槽位，避免 next_action_nodes 写入溢出。
    4. **评估所需接口补齐**：EventDrivenEnv 补充 `seed()`，支持 eval 固定种子。
*   **配置调整（L2）**：
    *   epsilon_by_stage.L2.decay_steps 改为 80000。
    *   L2 time_scale 与 churn_tol_override_sec 对齐 L1（0.8 / 1200）。
    *   L2 增加小剂量 potential shaping（alpha=0.05）。
*   **涉及文件**：`src/train/runner.py`、`src/env/gym_env.py`、`src/train/dqn.py`、`src/train/replay_buffer.py`

## 📚 训练协议规范

### EdgeQ Training Protocol 核心原则
1. **Rho-Gated Transitions**：阶段转换由 `rho` 阈值控制，而非步数超时。
2. **Epsilon Continuity**：Epsilon 仅依赖 `global_step`，跨阶段连续不重置。
3. **Reward Stability**：Phase3 奖励权重从 Phase2 线性插值，不回退默认值。
4. **Evaluation-Based Selection**：最佳模型由固定种子评估选出，非训练极值。
5. **State Continuity**：训练状态（replay/optimizer/RNG）跨 phase 连续。
6. **Audit Trail**：所有性能回退可从日志追溯。

**详见文档**：`docs/项目演进文档/training_protocol_edgeq.md`
