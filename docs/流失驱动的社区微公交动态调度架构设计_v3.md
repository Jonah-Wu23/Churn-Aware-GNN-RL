## **8\. 训练策略：结构化课程学习 (Structured Curriculum Learning)**

### **8.1 课程设计的核心哲学**

放弃全局均匀加密，采用\*\*结构化密度（Structured Density）\*\*生成，模拟真实世界的非均匀压力。

* **对抗中心诱饵：** 在中心区域堆积短途、高价值订单，诱惑 Agent 放弃边缘。  
* **保护边缘稀疏：** 在边缘区域放置长途、高流失风险订单，强迫 Agent 学会“延迟满足”。

### **8.2 课程推进机制 (The Trigger)**

如何判断 Agent 何时可以进入下一难度等级？不看单一的服务率，而是看\*\*“有效负荷率” (\\rho)\*\*。

* **Service Rate:** 成功完成的订单比例。  
* **Constraint Saturation (Stuckness):** 衡量 Agent 是否“动弹不得”。定义为：**被 Hard Mask 封锁的动作比例**。  
  * 如果 Saturation 高，说明 Agent 处于“频繁违约边缘”，此时虽未流失，但稍有扰动就会崩盘（过拟合边缘）。  
  * 如果 Saturation 低且 Service Rate 高，说明 Agent 游刃有余，可以“加量”。

### **8.3 课程阶段 (Curriculum Phases)**

| 阶段 | 场景特征 (Scenario) | 目的 (Goal) | 晋级条件 (Trigger) |
| :---- | :---- | :---- | :---- |
| **L0: 幼儿园** | 稀疏全局需求，无限 T\_{tol} | 学会连通性，跑通全图，不撞南墙 | Service \> 95% |
| **L1: 诱惑测试** | 中心高密度短单，边缘无单 | 学会利用容量，跑出最大营收 | \\rho \> 0.8 |
| **L2: 道德困境** | 中心高密度 \+ 边缘稀疏长单 | 学会**拒绝**部分中心单以保留余量给边缘 | \\rho \> 0.75 (且边缘服务率达标) |
| **L3: 真实世界** | 真实泊松分布 \+ 真实 T\_{tol} | 综合权衡流失博弈 | 收敛即可 |
| **L4: 压力测试** | 1.5倍高峰流量 \+ 事故扰动 | 验证鲁棒性 (Mask机制起效) | 观测崩溃点 |
